# Исправление адаптивной верстки

## Проблема

При открытии сайта на десктопе (>1024px) отображалась мобильная/планшетная версия, а при сжатии окна появлялась десктопная версия. Это происходило из-за неправильного использования Tailwind CSS breakpoints.

## Причина

Tailwind CSS по умолчанию использует **mobile-first** подход:
- `md:` означает "применить стили **начиная с** 768px и выше"
- `lg:` означает "применить стили **начиная с** 1024px и выше"

Когда мы писали:
```html
<nav class="flex md:fixed md:flex-col">
```

Это означало:
- **По умолчанию** (на всех экранах): `flex` (горизонтальная навигация)
- **От 768px и выше** (desktop): `fixed flex-col` (мобильное меню)

То есть на десктопе применялись мобильные стили!

## Решение

Используем **desktop-first** подход с `max-*` префиксами:

```html
<nav class="flex max-md:fixed max-md:flex-col">
```

Теперь это означает:
- **По умолчанию** (на всех экранах): `flex` (горизонтальная навигация)
- **До 767px** (mobile): `fixed flex-col` (мобильное меню)

## Изменения в tailwind.config.js

Добавлены кастомные max-width breakpoints:

```javascript
screens: {
  // Standard min-width breakpoints
  'sm': '640px',
  'md': '768px',
  'lg': '1024px',
  'xl': '1280px',
  '2xl': '1536px',
  
  // Max-width breakpoints for desktop-first approach
  'max-sm': {'max': '639px'},    // До 640px (small mobile)
  'max-md': {'max': '767px'},    // До 768px (mobile)
  'max-lg': {'max': '1023px'},   // До 1024px (tablet)
  'max-xl': {'max': '1279px'},   // До 1280px (small desktop)
}
```

## Таблица breakpoints

| Breakpoint | Экран | Применение |
|------------|-------|------------|
| Base (без префикса) | Все экраны | Десктопные стили |
| `max-sm:` | < 640px | Очень маленькие мобильные |
| `max-md:` | < 768px | Мобильные устройства |
| `max-lg:` | < 1024px | Планшеты |
| `md:` | ≥ 768px | От планшета и выше |
| `lg:` | ≥ 1024px | От десктопа и выше |

## Примеры исправлений

### Навигация

**Было (неправильно):**
```html
<div class="flex md:fixed md:flex-col">
```
- Desktop: fixed + flex-col (мобильное меню) ❌
- Mobile: flex (горизонтальная навигация) ❌

**Стало (правильно):**
```html
<div class="flex max-md:fixed max-md:flex-col">
```
- Desktop: flex (горизонтальная навигация) ✅
- Mobile: fixed + flex-col (мобильное меню) ✅

### Grid layout

**Было (неправильно):**
```html
<div class="grid grid-cols-3 md:grid-cols-1">
```
- Desktop: 1 колонка ❌
- Mobile: 3 колонки ❌

**Стало (правильно):**
```html
<div class="grid grid-cols-3 max-md:grid-cols-1">
```
- Desktop: 3 колонки ✅
- Mobile: 1 колонка ✅

### Кнопка меню

**Было (неправильно):**
```html
<div class="hidden md:block">
```
- Desktop: Показывается ❌
- Mobile: Скрыта ❌

**Стало (правильно):**
```html
<div class="hidden max-md:block">
```
- Desktop: Скрыта ✅
- Mobile: Показывается ✅

## Рекомендации для разработки

При добавлении новых адаптивных стилей:

1. **Начинайте с десктопа** - базовые классы без префиксов
2. **Используйте max-* для меньших экранов**
3. **Логика:**
   ```
   Базовый класс = Desktop
   max-lg: = Tablet
   max-md: = Mobile
   max-sm: = Small Mobile
   ```

### Пример правильного подхода:

```html
<!-- Правильно: Desktop-first -->
<div class="
  grid grid-cols-4 gap-8         <!-- Desktop: 4 колонки -->
  max-lg:grid-cols-2             <!-- Tablet: 2 колонки -->
  max-md:grid-cols-1 max-md:gap-4  <!-- Mobile: 1 колонка -->
">
```

### Чего НЕ делать:

```html
<!-- Неправильно: Mobile-first (базовые стили для мобильных) -->
<div class="
  grid grid-cols-1               <!-- Все: 1 колонка -->
  md:grid-cols-2                 <!-- От 768px: 2 колонки -->
  lg:grid-cols-4                 <!-- От 1024px: 4 колонки -->
">
<!-- Это работает, но не соответствует нашему desktop-first подходу -->
```
